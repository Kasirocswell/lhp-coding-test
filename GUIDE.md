# Coding Test Guide — Event Visuals

A practical guide to setting up, running, and reviewing this project. For the
*why* behind the design choices, see [`DECISIONS.md`](./DECISIONS.md); for the
original brief, see [`CODING_TEST.md`](./CODING_TEST.md).

---

## 1. What this is

A Laravel + Inertia + Vue 3 app that presents a seeded events dataset through
**two distinct browsing layouts**, with locally-served images, human-readable
addresses derived from coordinates, timezone-aware dates, filtering, RSVP, and
confirmation/reminder emails.

| Page | Route | Layout |
| --- | --- | --- |
| **Visual 1 — Gallery** | `/events-visual-1` | Image-forward responsive card grid |
| **Visual 2 — Timeline** | `/events-visual-2` | Chronological agenda grouped by day |
| **Event detail** | `/events/{event}` | Full info, image gallery, RSVP form |

`/` redirects to the gallery.

---

## 2. Prerequisites

- **PHP** 8.2+ with the usual Laravel extensions (`pdo_sqlite`, `mbstring`, `openssl`)
- **Composer** 2.x
- **Node.js** 18+ and **npm**
- SQLite (no separate database server needed)

Check versions:

```bash
php -v && composer -V && node -v
```

---

## 3. First-time setup

```bash
# 1. Install dependencies
composer install
npm install

# 2. Environment file + app key
cp .env.example .env
php artisan key:generate

# 3. Create the SQLite database
touch database/database.sqlite

# 4. Migrate and seed (smaller seed = fast local dev)
SEED_ROWS=4000 php artisan migrate --seed

# 5. Generate the local placeholder images
php artisan images:generate-placeholders
```

> **Seed size:** the default seed is ~1.25M rows to mirror real scale. For local
> review, `SEED_ROWS=4000` is plenty and seeds in seconds. Raise it if you want
> to stress-test performance.

---

## 4. Running the app

The simplest way runs the web server, a **queue worker** (for emails), and Vite
together:

```bash
composer dev
```

Then open **http://localhost:8000**.

> **Why the worker matters:** confirmation/reminder emails are pushed onto a
> database queue and sent by a background worker. If you start only
> `php artisan serve` without a worker, RSVP emails will sit unsent in the
> `jobs` table. `composer dev` includes the worker; if you run pieces manually,
> start one too:

```bash
php artisan serve          # web server
php artisan queue:work     # processes queued emails
npm run dev                # Vite dev server
```

## 5. Feature walkthrough (what to review)

### Two distinct layouts
- **Gallery** (`/events-visual-1`): cards crossfade to a second image on hover,
  lift on hover, and stagger-fade in on load.
- **Timeline** (`/events-visual-2`): events grouped by day with sticky date
  headers along a vertical rail.

### Images (end to end, served locally)
- Generated by `php artisan images:generate-placeholders` into
  `public/images/events/{type}/{1,2,3}.svg`.
- Each event is assigned a stable set of 3 images derived from its id — no
  external URLs, no giant image table.

### Addresses from lat/lng
- `app/Support/ReverseGeocoder.php` maps each coordinate to its nearest known
  city, giving a "City, Country" label plus an IANA timezone — fully offline.

### Timezone-aware dates
- Times display in the **venue's** timezone via `Intl.DateTimeFormat`. The
  detail page also shows **your local time** when it differs.

### Filtering
- Filter by **date range**, **location (city)**, plus search, category, status,
  and sort. Infinite scroll keeps it fast on the full dataset.

### RSVP + emails
- The detail page has a "Register your interest" form (Interested / Attending).
- Registering queues an **`AttendanceConfirmationMail`**.
- `events:send-reminders` (scheduled hourly) queues **3-day** and **24-hour**
  reminders; sent-at timestamps make it idempotent.

## 6. Email configuration

Mail is sent through **Resend** over the queue. Configure in `.env`:

```dotenv
MAIL_MAILER=resend
RESEND_API_KEY=your_resend_api_key
MAIL_FROM_ADDRESS="onboarding@resend.dev"
MAIL_FROM_NAME="Event Visuals"
```

Notes:
- `onboarding@resend.dev` works for testing but Resend will **only deliver to
  your own Resend account email** until you verify a custom domain. Set a
  verified domain as `MAIL_FROM_ADDRESS` to reach arbitrary recipients.
- For fully offline dev, set `MAIL_MAILER=log` to write emails to
  `storage/logs/laravel.log` instead of sending.

### Testing emails manually

```bash
# Trigger a confirmation: RSVP on any event detail page, then watch the worker.

# Fire the reminder pass on demand:
php artisan events:send-reminders

# If you ever run without a worker, drain the queue once:
php artisan queue:work --once --stop-when-empty
```

## 9. Troubleshooting

| Symptom | Likely cause / fix |
| --- | --- |
| RSVP email never arrives | No queue worker running. Use `composer dev`, or run `php artisan queue:work`. |
| Email "sent" but not received | Resend `onboarding@resend.dev` only delivers to your account email until a domain is verified. Check the Resend dashboard's Emails log. |
| No images show | Run `php artisan images:generate-placeholders`. |
| "no such table" errors | Run `php artisan migrate --seed` (and `touch database/database.sqlite`). |
| Slow first load on huge seed | Reduce `SEED_ROWS`, or rely on the date filter (gallery defaults to upcoming). |
| Reminders not firing | The scheduler must run: `php artisan schedule:work` (or trigger once with `php artisan events:send-reminders`). |
